Manatee Admin User Manual

This guide covers all Admin workflows in Manatee: setup, user management, matching oversight, email configuration, and audit/history.

1. Login and Access

1. Go to the login page.

2. Sign in with your Admin account.

3. You will land on the Admin Dashboard.

Default first-run credentials are admin/admin unless changed.

2. Dashboard Overview

The top cards show live totals:

1. Mentors.

2. Mentees.

3. Pending Requests.

4. Live Mentorships.

Each card is clickable and opens its detailed list page.

3. Managing Lookup Lists

Use dashboard sections to manage:

1. Locations.

2. Functions.

3. Mentorship End Reasons.

Each list supports add and delete actions. Changes are reflected immediately for users.

4. SMTP and Email Settings

Configure the system email sender in SMTP and Email Logging:

1. SMTP host, port, secure mode.

2. SMTP user and password.

3. From Email.

4. BCC Logging Email (copy of all outbound system emails).

After changes, use Test Settings to verify delivery acceptance by the SMTP server.

5. Users Page

Open Users to view all registered users.

For each user you can see role, active relationship counts, and creation time.

Admins can delete non-admin users. Deletion removes linked data through relational cleanup rules.

6. Pending Requests

Open Pending Requests to view all mentorship requests waiting for mentor action.

Admin can resend a request reminder to the mentor on behalf of the mentee.

7. Live Relationships

Open Live Relationships to view all accepted mentorship pairs currently active.

8. Reset Mentor Relationships

Use Reset Mentor Relationships to archive non-ended records for one selected mentor.

This does not erase ended history; it marks active/pending records as ended with an Admin reset reason.

9. Relationship History and Export

Relationship History shows historical mentorship lifecycle records.

Use Export CSV to download history for reporting or compliance.

10. Audit Log

Audit Log shows email/audit events such as sends, failures, SMTP test outcomes, and workflow notifications.

Use filters for event type, status, recipient, and date range.

11. Admin Best Practices

1. Keep From Email aligned with SMTP authenticated sender unless your mail platform supports verified aliases.

2. Use SMTP test before going live.

3. Review audit logs frequently for failed emails.

4. Export relationship history regularly if required by policy.
