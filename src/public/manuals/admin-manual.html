Manatee Admin User Manual

Welcome to Manatee administration. This guide covers dashboard operations, user management, mentorship lifecycle oversight, system configuration, and reporting tools.

Getting Started

Logging In

  • Navigate to the login page and enter your admin username and password
  • If two-factor authentication (2FA) is enabled on your account, enter the code from your authenticator app
  • You will be directed to the Admin Dashboard

Default Credentials: The initial admin account uses username admin with password admin. Change this immediately upon first login.

Dashboard Overview

The Admin Dashboard displays four key metrics at the top:

  • Total Mentors: Click to view all mentor accounts
  • Total Mentees: Click to view all mentee accounts
  • Pending Requests: Click to view open mentorship requests awaiting mentor response
  • Active Mentorships: Click to view currently accepted mentor-mentee pairs

Each metric card is interactive—click any card to drill into the detailed list view for that category.

User Management

Viewing and Editing Users

Access Users from the dashboard to see all registered accounts. For each user, the list displays:

  • Username and email address
  • Role (Admin, Mentor, Mentee, or Both)
  • Count of active mentorships
  • Account creation and last login timestamps

User Actions

  • Create Admin User: Only available to superusers. Creates a new admin account with elevated permissions
  • Resend Welcome Email: Sends or resends the welcome/onboarding email to a user
  • Send Password Reset: Issues a password reset link that the user can access from the login page
  • Delete User: Removes a user account. Use caution—this may cascade to mentorships and history records depending on your retention policy

Mentorship Lifecycle Management

Pending Requests

View all open mentorship requests waiting for mentor action in Pending Requests. Each request shows:

  • Request ID and creation date
  • Mentee name and location
  • Requested function/specialty
  • Status and last update time

You can resend a request reminder to the mentor on behalf of the mentee if needed.

Live Relationships

View all currently active, accepted mentorships in Live Relationships. From this page, you can:

  • View relationship start dates and current status
  • Manually end a relationship (e.g., if urgent termination is needed) by selecting a reason
  • Monitor relationship progress and history

Relationship History

Relationship History contains all ended or archived mentorship records. Sort and filter by:

  • Relationship ID, Status
  • Mentor and Mentee names
  • Location, Function, End Reason
  • Who ended the relationship and when

Use Export CSV to download history for reporting, compliance, or external analysis.

Reset Mentor Relationships

If a mentor needs to have all their active/pending relationships cleared, use Reset Mentor Relationships to:

  • Select a specific mentor
  • Archive all non-ended records as "Admin Reset"
  • Preserve full history (nothing is deleted)

Reference Data Management

Maintain the lookup lists that users depend on. Access Locations, Functions, and Mentorship End Reasons from the dashboard.

List Operations

  • Add: Click the input box and type a new entry, then press Enter or click Add
  • Edit: Click any existing entry to edit it inline, then press Enter to save
  • Delete: Remove entries using the delete button (only available if the entry is not in use)

CSV Import

Bulk load data by uploading a CSV file. The system performs automatic validation:

  • Blank rows are skipped
  • Duplicate rows within the file are deduplicated
  • Rows that already exist in the database are skipped
  • Changes are reflected immediately for all users

System Configuration

Theme and Branding

Control the visual appearance of Manatee in Theme Settings:

  • Primary Color: Main accent color used throughout the interface
  • Secondary Color: Highlight color for unread goal badges and notifications
  • Background Color: Page background
  • Logo Upload: Upload your organization's logo (PNG or JPG, recommended size 200×50px or larger)

SMTP and Email Configuration

Configure outbound email in SMTP Settings:

  • SMTP Host: Your mail server address (e.g., smtp.gmail.com, smtp.office365.com)
  • SMTP Port: Port number (typically 25, 587, or 465)
  • Secure Mode: Enable TLS or SSL as required by your mail server
  • Username & Password: Authentication credentials for your SMTP account
  • From Email: The sender email address that appears in all outbound messages
  • BCC Logging Email (optional): A copy of every email sent will be BCC'd to this address for audit/compliance

After entering SMTP settings, click Test Settings to verify that the server accepts your configuration. A test email will be sent to the "From Email" address.

Auditing and Compliance

Audit Log

The Audit Log records all system events including:

  • Email sends and delivery status (success or failure)
  • SMTP configuration test results
  • User account activity (login, password reset, etc.)
  • Mentorship workflow changes (requests, acceptances, endings)

Filter the log by:

  • Event type (email, auth, workflow, etc.)
  • Status (success, failure, pending)
  • Recipient email or user
  • Date range

Use the audit log to troubleshoot issues, verify email delivery, and maintain compliance records.

Administrator Best Practices

  • Security: Change the default admin password on first login. Use a strong, unique password and consider enabling 2FA.
  • Email Configuration: Test SMTP settings in a staging environment before deploying to production. Ensure the "From Email" matches your SMTP authenticated sender (many mail providers require this).
  • Reference Data: Keep Locations, Functions, and End Reasons up to date. Invalid data discourages user adoption.
  • Monitoring: Review the Audit Log weekly for email failures or unusual activity. Investigate failed emails immediately.
  • Backups & Exports: Regularly export relationship history for backup and compliance purposes, especially if your retention policy requires historical records.
  • User Lifecycle: When removing a user, understand the cascade impact on their mentorships and goal records.
  • Branding: Upload a high-quality logo and test theme colors across different browsers to ensure consistent appearance.

Troubleshooting

Emails Not Sending

  • Check SMTP settings in Theme Settings → SMTP Configuration
  • Click "Test Settings" to verify the connection
  • Review the Audit Log for error messages
  • Verify that the "From Email" is allowed by your SMTP provider

User Cannot Log In

  • Verify the user exists in the Users list
  • Send a password reset email
  • If 2FA is causing issues, ask the user to verify their authenticator app is synchronized

Mentorship Request Stuck

  • Check Pending Requests list
  • Verify the mentor's availability is turned on
  • Manually resend the request reminder via the admin interface
